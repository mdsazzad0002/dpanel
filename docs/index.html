Designed for practical hosting work

Each layer has a clear job, so developers can improve the panel without mixing UI code, privileged host actions, and installer recovery scripts.

W
Website Management

Create website roots, sync virtual hosts, choose PHP versions, and repair stack config.

A

Repeatable commands help recover permissions, vhosts, web stack services, and runtime config.

First install

Fresh servers usually need sudo before cloning into `/var/www`.

On many servers, `/var/www` is owned by root. The recommended command uses `sudo git clone`, then gives the cloned source back to your current user.

install 
sudo mkdir -p /var/www
cd /var/www
sudo git clone https://github.com/mdsazzad0002/dpanel.git dpanel-source
sudo chown -R "$USER:$USER" /var/www/dpanel-source
cd dpanel-source
sudo ./installer.sh chain install

Alternative: clone in your home directory, then run the installer with sudo.

Permission repair is built in

When the file manager can open a folder but cannot write, PHP and the website owner need shared write access.

Common first-run symptom The panel opens `/home/example/public_html`, but create, edit, upload, unzip, or delete fails with a permission denied error.
repair-permissions 
/var/www/dscript/scripts/fix-permissions.sh --all

# or repair one website
/var/www/dscript/scripts/fix-permissions.sh --path /home/example/public_html
