# phpMyAdmin SPA Enhancement Plan

## Goal
Make phpMyAdmin page professional/futuristic, eliminate full-page reloads, reduce all reload behavior to SPA-style state transitions.

## Root Cause of Full-Page Reloads
In `Index.vue`, `visitPhpMyAdminPage()` uses `router.visit()` (Inertia full page reload) for:
- **`handleToolbarAction`**: SQL, browse, structure, insert, operations, search
- **`handleOverviewSelect`**: SQL, Transfer, Databases, Status, User accounts, Settings, Replication, Variables, Charsets

The composable `usePhpMyAdminReadState.js` already has pure state-management versions (`handleToolbarAction`, `handleOverviewSelect`) that DON'T reload — but `Index.vue` overrides them with `router.visit()` versions.

---

## Step 1: Eliminate Full-Page Reloads in Index.vue
**File**: `resources/js/Pages/PhpMyAdmin/Index.vue`

### Remove `visitPhpMyAdminPage()` — replace with pure state transitions

**`handleToolbarAction` rewrite:**
| Action | Current (reload) | New (SPA) |
|--------|-----------------|-----------|
| `sql` | `router.visit()` | `overviewMode.value = 'sql'` |
| `browse` | `router.visit()` | `readToolbarAction('browse')` via composable |
| `structure` | `router.visit()` | `readToolbarAction('structure')` |
| `insert` | `router.visit()` | `readToolbarAction('insert')` |
| `operations` | `router.visit()` | `readToolbarAction('operations')` |
| `search` | `router.visit()` | `readToolbarAction('search')` |
| `import/export` | `router.visit()` | `overviewMode.value = 'transfer'` |
| `create` | `router.visit()` | `schemaEditorMode = 'create'` |

**`handleOverviewSelect` rewrite:**
| Tab | Current (reload) | New (SPA) |
|-----|-----------------|-----------|
| `SQL` | `router.visit()` | `overviewMode.value = 'sql'` |
| `Transfer` | `router.visit()` | `overviewMode.value = 'transfer'` |
| `Databases` | `router.visit()` | `overviewMode.value = 'databases'` |
| `Status` | `router.visit()` | `pushToast('Status page coming soon', 'info')` |
| `User accounts` | `router.visit()` | `pushToast('User accounts coming soon', 'info')` |
| `Settings` | `router.visit()` | `pushToast('Settings page coming soon', 'info')` |
| `Replication` | `router.visit()` | `pushToast('Replication page coming soon', 'info')` |
| `Variables` | `router.visit()` | `pushToast('Variables page coming soon', 'info')` |
| `Charsets` | `router.visit()` | `pushToast('Charsets page coming soon', 'info')` |
